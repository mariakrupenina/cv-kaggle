На локальной валидации всё выглядело так, будто модель сейчас разнесёт лидерборд, а на public score Kaggle вежливо попросил приземлиться :(

### Задача object detection на 52 класса для Kaggle-соревнования




------------------
Сначала я посмотрела структуру данных, проверила train/test, количество картинок и файлов с разметкой. В train было 1697 изображений, в test — 424 изображения. Разметка была в формате YOLO: class x_center y_center width height. Также я проверила, что всего 52 класса, классы идут от 0 до 51, пустых label-файлов и битых строк нет.

После этого я визуально проверила разметку: вывела несколько картинок с bounding boxes, чтобы убедиться, что рамки действительно совпадают с объектами. Все изображения были размера 640x640, поэтому для обучения использовался imgsz=640.

Дальше я подготовила данные для Ultralytics YOLO

Для обучения пробовала модели yolo11s и yolo11m. Лучше себя показала yolo11m

После обучения был сделан inference на test-изображениях. Предсказания модели были переведены в формат Kaggle

также подбирался confidence threshold

Лучший полученный public score: 0.33973.

В ноутбуке есть основные этапы: проверка данных, небольшой EDA, визуальная проверка разметки, подготовка YOLO-датасета, обучение модели, финальное обучение на всём train, inference, postprocessing и сохранение submission-файла.
